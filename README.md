The program is organized as a layered verification pipeline. In each run, the main routine first generates a structured random DES plant and immediately checks its nominal prognosability, so that only plants with prog = 1 are kept for further analysis. It then assigns an attack scenario by selecting insertion and erasure event sets, and constructs an attack-aware verifier to test robust prognosability against all attacks. In parallel, a reduced stealthy verifier is built to evaluate robustness against stealthy attacks only. To keep the generated examples meaningful, the program filters out oversized or uninformative instances and records, for every accepted run, the plant size, attack-set size, verifier size, stealthy-verifier size, and the corresponding computation times. In this way, the code separates plant generation, attack modeling, verifier construction, stealthy reduction, and result collection into distinct modules, which makes the whole procedure easy to adjust for different complexity ranges and attack scenarios
The design relies on several important parameters. The parameter numRuns specifies the total number of benchmark instances to be generated, while maxTriesPerRun limits the number of regeneration attempts for each run. The function bucket_config controls the structural complexity of the generated plant through parameters such as N_range, ambProb, returnNormalProb, and extraEdgeProb, which respectively determine the state-size range, the probability of expanding the ambiguous region, the probability of returning to the normal region, and the density of extra transitions. The parameter group used in generate_attack_scenario_guided determines whether the attack is labeled as benign, medium, or critical, and thus controls the aggressiveness of the insertion and erasure event sets. In addition, verifier-size constraints are imposed so that overly small, trivial, or excessively large instances are discarded before being written into the final dataset. This modular design makes it possible to jointly regulate plant complexity, attack strength, verifier size, and running time, thereby producing a benchmark set that is both logically informative and computationally representative.
